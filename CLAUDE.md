# CLAUDE.md — ppt-view 프로젝트 작업 규칙

## 발표 자료 작성 워크플로우

발표(`presentations/<slug>/`)를 만들 때는 **2단계**로 진행한다.

### 1단계 — `DRAFT.md` (슬라이드 내용 초안)

- 슬라이드 구성은 **marp 형식**으로 `presentations/<slug>/DRAFT.md` 에 작성한다.
- `---` 로 시작하는 `marp: true` frontmatter 로 열고, 슬라이드는 `---` 로 구분한다.
- **내용만 쓴다**: 제목 + 불릿/이미지 + 핵심 한 줄. 사용자가 읽으며 내용을 수정하는 용도다.
- 빌드 지침(섹션 라벨, "카드 N개", "KEY IDEA 박스", 이미지 배치 맵, 그라데이션 규칙 등 *어떻게 구성해라*)은 본문 텍스트로 **DRAFT.md 에 적지 않는다.** 그런 내용은 index.html 빌드 시점에 디자인 가이드에서 가져온다. 다만 특정 슬라이드·요소에 대한 개별 지시가 필요하면 본문이 아니라 `<!-- @claude: ... -->` 주석으로 남긴다(2단계 참조).
- 이미지는 `![](./assets/...)` 상대경로로 넣는다. 한 슬라이드에 이미지 여러 개 OK.

### 2단계 — `index.html` (디자인 적용 빌드)

- DRAFT.md 의 슬라이드 구성 그대로, `design-guide/design-white-navy.md` (또는 지정 테마) 디자인 시스템을 적용해 `index.html` 을 만든다.
- **`marp: true` 인 DRAFT 의 `---` 페이지 구분을 항상 그대로 지킨다.** 한 `---` 구획 = 슬라이드 한 장. 어떤 디자인 양식을 쓰든, 디자인 가이드가 "밀도가 높으면 나눠라" 같은 제안을 하더라도, 사용자가 `---` 로 나눈 슬라이드의 **수와 순서를 임의로 쪼개거나 합치지 않는다.** 최종 슬라이드 장수는 항상 DRAFT 의 `---` 구획 수와 일치해야 한다.
- **DRAFT 가 `![]()` 로 참조한 이미지만 넣는다.** 어떤 이미지를 어느 슬라이드에 넣을지는 오로지 DRAFT 의 `![]()` 링크가 정한다. `assets/` 폴더에 파일이 있다는 것만으로는 배치 근거가 되지 않는다 - DRAFT 가 참조하지 않은 이미지는 내용상 관련 있어 보여도, 그 자산이 안 쓰이게 되더라도 어느 슬라이드에도 넣지 않는다(목차 대표 그림, 재사용 표지, "예시용" 다이어그램 등 일절 금지).
- **`<!-- @claude: ... -->` 주석은 나에게 주는 빌드 지시다(내용 아님).** DRAFT 에 이 형식의 HTML 주석이 있으면 빌드 지시로 인식해 **반영한 뒤 출력에서 제거한다** - index.html 에 절대 렌더하지 않고 `※` 각주로도 만들지 않는다. **어느 슬라이드든 원하는 위치에 자유롭게 쓴다** - 위치가 사용을 제한하지 않고, 그 지시가 "무엇에 대한 것인지"만 정한다: `---` 구획 안에 있으면 그 슬라이드 전체, 특정 줄(이미지·불릿) 바로 아래면 그 요소, 첫 슬라이드 앞(frontmatter 근처)이면 덱 전체. 디자인 기본값과 충돌하면 **지시가 우선**한다. `@claude:` 키워드가 없는 일반 주석은 지시가 아니므로 그대로 무시한다.
- **`※` 로 시작하는 문장은 주석 처리한다(테마 무관).** DRAFT 에서 `※` 로 시작하는 문장(저자의 부연·정의·근거 메모)은 본문보다 작고 흐린 주석 스타일 - 대략 11~12px, 흐린 회색(muted), `※` 기호 유지, 걸이 들여쓰기(hanging indent) - 로 렌더한다. **배치는 이미지에 붙었는지로 갈린다**: `![]()` 이미지 **바로 다음 줄**(빈 줄 없이)에서 `※` 가 시작하면 그 이미지의 주석이므로 **그림 바로 아래에 붙여** 둔다. 그 외(단독 주석, 또는 이미지와 빈 줄로 떨어진 경우)는 **슬라이드 맨 밑 가장자리 각주로 통일**하고, 그 위에 얇은 hairline 구분선(약 1px, 텍스트 칼럼 폭 정도)을 둔다. 박스·테두리 없이 작고 흐린 글씨 자체가 주석 신호다. 어떤 테마를 쓰든 항상 적용한다.
- **문단(그룹) 사이 여백 vs 리스트 내부 간격을 구분한다.** 한 슬라이드에서 서로 다른 문단·라벨 그룹 사이는 넉넉히(대략 24~28px), 같은 리스트 안의 불릿(`-`/`-`) 사이는 촘촘히(약 12~16px) 둔다. 여백은 아이디어를 나누는 신호이지, 한 리스트를 흩뜨리는 용도가 아니다.
- 빌드 요청 표준 문구: **"DRAFT.md 의 슬라이드 구성대로 design-guide 디자인 시스템 사용해서 index.html 만들어줘."**
- 테마 토큰은 `/shared/themes/<theme>.css` 로 자동 연결되고, 폰트/PDF 인쇄 스타일은 dev `/p` 라우트가 자동 주입한다. 사이드바·KEY IDEA·카드 등 레이아웃 컴포넌트는 index.html 에 직접 구현한다.

## 로컬 미리보기

- dev 서버는 포트 3000 을 피한다(다른 앱 점유). `npm run dev -- -p 3100` 처럼 다른 포트로 띄운다.
- 미리보기: `http://localhost:<port>/view/<slug>`

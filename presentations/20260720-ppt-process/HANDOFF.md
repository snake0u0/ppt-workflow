# 결과물 요건 및 ppt-view 역이전 규칙

Claude Design 에서 만든 발표자료를 최종적으로 이 프로젝트(`ppt-view`)로 다시
가져와 배포한다. 이 문서는 그 전제에서 **어떤 형태로 나와야 옮기기 쉬운지**를
정리한 것이다. 디자인(색·타이포·레이아웃)은 Claude Design 쪽 테마를 따르며,
여기서 지정하지 않는다.

---

## 1. 결과물 요건

옮겨올 때 손이 가장 적게 드는 형태다.

- **단일 `index.html`.** CSS 는 `<style>` 로, JS 는 `<script>` 로 문서 안에 인라인.
  외부 CDN 링크(폰트, 스크립트, 이미지) 없이 자체 완결되어야 한다.
- **논리 캔버스 1280x720 (16:9).** 슬라이드는 이 크기로 고정해서 그리고, 화면
  크기에는 `transform: scale()` 로 맞춘다. 뷰포트 단위(`vw`/`vh`)로 본문 크기를
  잡지 않는다. 인쇄 시 슬라이드 1장 = 용지 1장이 되어야 한다.
- **슬라이드 전환은 표시/숨김 방식.** 슬라이드 23장을 모두 DOM 에 두고 활성화된
  한 장만 보이게 한다. 세로 스크롤로 이어지는 문서형은 안 된다.
  ```html
  <div id="deck">
    <section class="slide active">...</section>
    <section class="slide">...</section>
  </div>
  ```
  ```css
  .slide { position: absolute; inset: 0; display: none; }
  .slide.active { display: block; }
  ```
- **좌우 방향키로 넘길 수 있게.** 이 프로젝트의 다른 덱과 조작이 같아야 한다.
- **이미지 경로는 `./assets/파일명`** 상대경로. base64 로 인라인하지 않는다
  (파일이 비대해지고 이미지 교체가 불가능해진다).
- **수식은 KaTeX/MathJax 로 렌더.** 수식을 이미지로 구워서 넣지 않는다. 나중에
  수정할 수 없게 된다.
- **한글 폰트를 반드시 폰트 스택에 둔다.** Inter 같은 라틴 전용 폰트만 지정하면
  한글이 조용히 시스템 기본 폰트로 떨어져 디자인이 깨진다. 라틴 폰트를 쓰더라도
  `'Inter', 'Pretendard', system-ui, sans-serif` 처럼 뒤에 한글 폰트를 붙인다.
- **`<!-- @claude: ... -->` 지시는 결과물에 남지 않아야 한다.** 반영하고 지운다.

## 2. 옮겨오기

### 2-1. 폴더 구성

이 폴더(`presentations/20260720-ppt-process/`)의 구성은 다음과 같이 유지한다.

```
presentations/20260720-ppt-process/
  assets/       첨부 이미지·영상 (그대로 둔다)
  DRAFT.md      발표 원고 - 사람만 수정한다
  index.html    실제 발표자료 - AI가 수정한다
  meta.json     메타데이터
```

- **`DRAFT.md` 를 결과물에 맞춰 고치지 않는다.** 원고가 원본이고 `index.html` 이
  파생물이다. 방향이 반대가 되면 원고가 AI 출력으로 오염된다.
- 받아온 HTML 은 `index.html` 이라는 이름으로 저장한다.
  (`slide.html`, `deck.html` 등 다른 이름 금지.)
- `assets/` 는 건드리지 않는다. 이미 다 들어있다.

### 2-2. 폰트

폰트 파일은 저장소 루트 `fonts/<폰트명>/` 에 두면 `npm run gen` 이 `@font-face`
CSS 를 만들고, 뷰어가 `/shared/fonts.css` 를 자동 주입한다. 현재 들어있는 것은
Pretendard, Inter, Paperlogy 세 가지다.

**Claude Design 테마가 이 목록에 없는 폰트를 쓴다면** 폰트 파일을
`fonts/<폰트명>/` 에 추가하고 `npm run gen` 을 한 번 돌린다. 그러지 않으면
로컬에서만 보이고 배포 후에는 폰트가 빠진다.

### 2-3. `meta.json`

```json
{
  "title": "AI를 활용한 발표자료 제작 워크플로우",
  "date": "2026-07-20",
  "event": "labmeeting",
  "theme": "<Claude Design 테마 이름>"
}
```

`theme` 값은 현재 **기록용 메타데이터**다. 코드가 읽어서 CSS 를 붙이지 않는다.
`/shared/themes/*.css` 는 `design-guide/design-*.md` 가 있는 테마에 대해서만
생성되므로, Claude Design 테마에는 대응 파일이 없다. 따라서:

- `index.html` 에 `<link rel="stylesheet" href="/shared/themes/...">` 를 **넣지
  않는다.** (없는 파일을 가리키게 되고, 색 토큰이 전부 빈 값이 된다.)
- 색·타이포 토큰은 `index.html` 안 `:root` 에 직접 선언한다.

### 2-4. 뷰어가 자동으로 주입하는 것

`index.html` 에 직접 넣을 필요가 없다. 뷰어가 `<head>` 에 끼워 넣는다.

- `/shared/fonts.css` (폰트)
- `/deck-print.css` (인쇄용, 슬라이드 1장 = 1페이지)
- `/lightbox.js` (이미지 확대)

**주의:** 주입 로직은 HTML 에 `"/deck-print.css"` 라는 문자열이 이미 있으면
"주입 완료"로 보고 건너뛴다. 그래서 주석에라도 이 문자열을 적어두면 폰트와 인쇄
스타일이 통째로 빠진다. `index.html` 안에서 이 경로를 언급하지 않는다.

### 2-5. 확인 후 배포

```bash
npm run dev -- -p 3100     # 포트 3000은 다른 앱이 쓰는 중이라 피한다
```

1. `http://localhost:3100/view/20260720-ppt-process` 에서 확인
2. **슬라이드가 23장인지 센다.** (원고 `---` 구획 수와 같아야 한다)
3. 겹침·잘림·넘침 확인. 특히 이미지가 여러 장 한 행에 들어간 슬라이드
   (5, 6, 7, 8번)는 캔버스 폭 1280px 을 넘기기 쉽다.
4. 영상 슬라이드(18번) 재생 확인
5. 인쇄 미리보기로 PDF 변환 확인
6. 썸네일 갱신: `npm run gen-thumbs`
   (이 스크립트는 다른 덱 썸네일까지 다시 굽는다. **이 덱의 `.webp` 만
   스테이징**하고 나머지는 되돌린다.)
7. main 에 커밋·푸시 → Vercel 자동 배포

## 3. 체크리스트

옮긴 뒤 아래가 전부 참이어야 한다.

- [ ] 슬라이드 23장, 원고와 순서 동일
- [ ] 원고 문장·목차·제목이 한 글자도 바뀌지 않음
- [ ] 원고가 참조한 자산만 등장 (미참조 4개 파일은 어디에도 없음)
- [ ] `<!-- ... -->` 발표자 노트가 화면에 보이지 않음
- [ ] `<!-- @claude: -->` 지시가 결과물에서 제거됨
- [ ] `※` 줄이 작고 흐린 각주로 렌더 (본문 크기로 섞이지 않음)
- [ ] 한글이 의도한 폰트로 렌더 (시스템 기본 폰트로 떨어지지 않음)
- [ ] 이미지 경로가 `./assets/` 상대경로
- [ ] 인쇄 시 슬라이드 1장 = 1페이지
